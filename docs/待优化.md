### 情报类别同步到 H5 日历（2026-04-14）

**改动文件**:
- `server/app/api/h5_intel.py` — `"type": intel.category`
- `h5/src/pages/calendar/index.tsx` — 过滤栏改为 7 个类别，移除 expo/merch/sale

## 待处理

**优先级**: 中  
**模块**: 微博情报 / AI 批次提取  
**日期**: 2026-04-14

### 问题描述

当前 `server/app/services/weibo_intel_service.py` 中的 AI 批次提取提示词（`CATEGORY_SYSTEM_PROMPT` 和 `BATCH_CATEGORY_SYSTEM_PROMPT`）将情报类别硬编码在 Prompt 模板中，共 7 种固定类型：

| 值 | 名称 |
|---|---|
| convention | 漫展 |
| book_signing | 签售 |
| pre_order | 预售 |
| product_launch | 新谷开团 |
| offline_activity | 线下活动 |
| online_activity | 线上活动 |
| other | 其他 |

这些类别同时硬编码在：
- `server/app/models/weibo_intel.py` 的 `IntelCategory` 枚举
- `server/app/services/weibo_intel_service.py` 的 Prompt 模板
- `manager_web/src/types/weiboIntel.ts` 的类型定义和 `INTEL_CATEGORY_MAP` 映射

### 影响

- 要新增/修改/删除类别，需要同时修改后端 Python 代码、前端 TypeScript 代码和 Prompt 模板，**多处维护，容易遗漏**
- 无法通过管理界面灵活调整类别
- Prompt 模板中的类别说明与代码不同步

### 优化方向

1. **数据库化**: 将类别定义为数据库中的配置数据，支持 CRUD 管理界面
2. **统一管理**: 后端 API 提供类别列表接口，前后端共用同一份定义
3. **Prompt 动态生成**: 批次提取时从数据库读取类别配置，动态拼接到 Prompt 中
4. **类型安全**: 前端通过 API 获取类别列表，避免硬编码

### 相关文件

- `server/app/models/weibo_intel.py` — IntelCategory 枚举定义
- `server/app/services/weibo_intel_service.py` — Prompt 模板（第 64-133 行）
- `manager_web/src/types/weiboIntel.ts` — 前端类型定义和映射
