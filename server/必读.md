## 一、产品定位与商业逻辑

### 1.1 核心场景

**二次元展会 + 谷子（ACG周边）情报聚合平台**

- 用户通过本平台一站式获取谷子新品、展会资讯、限购通知
- 无需逐个关注官方微博，通过本平台即可聚合浏览

### 1.2 目标用户

- 二次元爱好者、谷子收藏者、展会参与者
- 需要快速获取官方公告信息的用户

### 1.3 变现模式

- **阿里妈妈 CPS 推广**：平台接入阿里妈妈联盟，用户通过本平台点击跳转购买谷子，创作者获得佣金
- **付费会员服务**：未来可能提供会员专属功能
- **禁止行为**：禁止贩卖原始微博数据、禁止将抓取的数据用于第三方数据交易

### 1.4 信息来源的合理性

二次元展会和谷子圈天然围绕一批固定的官方账号运转：

- 展会主办方（CP、ChinaJoy、BW 等）
- 动漫 IP 官方微博
- 谷子品牌/官方店铺
- 同人展会组织

**这和娱乐记者追踪固定信源没有区别，信息来源高度可预期、账号列表稳定。**

## 二、数据规模约束

### 2.1 用户规模

- **微博账号数量：不超过 100 个**
- 所有待监控账号必须经过筛选，确保是官方/权威账号
- 禁止无限制扩展账号数量

### 2.2 更新频率

| 参数 | 值 | 说明 |
|------|-----|------|
| 用户间隔 | 4 秒 | 每个账号抓取间隔必须 ≥ 4s |
| 全文请求间隔 | 3 秒 | 长文本接口请求间隔必须 ≥ 3s |
| 更新窗口 | 3 天 | 只抓取 3 天内发布的帖子全文 |
| 每用户每日上限 | 50 条 | 每个账号每天最多爬取 50 条微博 |

### 2.3 数据保留

- 纯文本内容 + 互动数据（点赞/评论/转发数）
- **不存储图片、视频**
- 自动清理旧数据，保留最新 100 条

### 2.4 禁止抓取的内容

- 用户私信
- 关注列表/粉丝列表
- 评论内容（仅统计数量）
- 图片和视频文件
- 任何隐私相关信息

## 三、技术实现约束

### 3.1 请求方式

- **复用浏览器已登录的 cookie**（用户自己登录的微博账号）
- 调用微博内部 AJAX API（非官方公开 API）：
  - `https://weibo.com/ajax/statuses/mymblog` — 获取用户微博列表
  - `https://weibo.com/ajax/statuses/longtext?id={mblogid}` — 获取长文本全文
- 请求必须携带正确的 cookie 和 User-Agent

### 3.2 频率控制

- 用户间隔 ≥ 4 秒
- 全文请求间隔 ≥ 3 秒
- 必须实现增量更新逻辑（避免重复抓取已有数据）
- 必须实现登录状态检测（cookie 过期时自动告警并停止）

### 3.3 禁止的技术行为

- ❌ 使用代理 IP 池绕过 IP 限制
- ❌ 伪造或随机切换 User-Agent
- ❌ 绕过验证码或人机验证
- ❌ 暴力请求导致接口限流
- ❌ 大规模批量请求（> 100 账号）
- ❌ 使用非本人账号的 cookie
- ❌ 尝试破解微博反爬机制

### 3.4 登录状态管理

- 浏览器 cookie 存储在本地配置文件
- 必须有登录过期检测机制
- 检测到登录过期时，立即停止爬虫并告警
- 不自动尝试重新登录或刷新 cookie

## 四、法律风险与合规边界

### 4.1 综合法律风险评估

**结论：低风险**

### 4.2 风险维度分析

| 维度 | 风险等级 | 说明 |
|------|----------|------|
| 微博用户协议 | 中 | ToS 通常禁止自动化抓取，但低频自有账号使用不在重点打击范围 |
| 个人信息保护 | 低 | 仅抓取公开帖子，不涉及隐私数据 |
| 账号安全 | 低 | 频率控制合理，不会触发风控 |
| 版权风险 | 极低 | 仅抓取文字，不复制图片/视频 |
| 商业用途 | 低 | 变现靠 CPS 和自服务，不卖数据 |
| 自动化强度 | 低 | 行为模式接近人工浏览 |

### 4.3 微博重点打击的行为

以下行为**绝对禁止**，会导致法律风险：

- 大规模批量抓取（数千至数万个账号）
- 绕过反爬机制（IP 封禁、验证码等）
- 盗取用户隐私数据
- 商业化贩卖原始数据
- 使用非本人账号

### 4.4 本项目的合规性依据

1. ✅ **抓取公开内容**：展会公告、限购通知本身就是要公开的信息
2. ✅ **不使用图片/视频**：完全避免媒体文件合规问题
3. ✅ **固定小规模**：100 个账号属于正常信息追踪范围
4. ✅ **商业变现走正道**：阿里妈妈 CPS + 自服务，不卖数据
5. ✅ **账号是自己的**：不盗用他人微博账号
6. ✅ **频率克制**：4 秒间隔接近正常用户行为

### 4.5 如需调整的场景

如果未来需要扩展功能，请先评估以下问题：

1. 新增账号是否仍然属于"固定信源"范畴？
2. 数据用途是否仍然限于"聚合展示"而非"数据交易"？
3. 请求频率是否仍然克制？
4. 是否仍然只抓取公开内容？

**如有疑问，请先讨论确认再行动。**

## 五、开发原则

### 5.1 核心原则

1. **克制优先**：宁可少抓，不可过度
2. **增量更新**：永远只抓新内容，不重复抓取已有数据
3. **防呆设计**：登录过期自动停止，频率不够自动等待
4. **可追溯**：所有抓取行为记录日志，便于排查问题
5. **不越界**：不在未讨论确认的情况下新增功能

### 5.2 性能目标

- 单次任务处理 100 个账号，总耗时约 6-8 分钟
- 数据库存储量控制在合理范围
- 浏览器资源占用最小化

### 5.3 监控与告警

- 任务状态实时可见（运行中/暂停/完成/失败）
- 登录状态异常自动告警
- 抓取失败记录日志并继续处理下一个账号
