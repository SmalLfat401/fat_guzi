> 最后更新：2026-03-29

## 项目概述

Fat OpenClaw 是一个面向二次元/谷圈用户的内容平台，核心价值：**帮助用户了解谷圈黑话 + 找到心仪的谷子商品**。

### 核心数据

- **术语库**：4大分类（谷子/Coser/漫展/游戏）+ AI 文案生成
- **商品库**：谷子商品搜索 + 阿里妈妈返佣链接
- **内容源**：微博 KOL 爬虫系统
- **技术栈**：Python FastAPI + MongoDB + Playwright（后端）| React + Ant Design（管理后台）

### 平台路线

```
运营管理后台（已完成）
        ↓
┌─ 社群情报（引流工具）─────────────┐
│  微信公众号 + 微博监控 + 微信群提醒  │
└──────────────────────────────────┘
        ↓
微信小程序 MVP（待开发）→ 抖音/支付宝小程序
        ↓
用户沉淀 + 内容引流 + 用户增长
        ↓
四大增值业务变现
        ↓
工商个体户 → 接入小程序支付
```

#### 3.3.3 事件去重与聚合

**事件指纹算法：**

```python
def generate_event_fingerprint(text: str, core_keywords: list) -> str:
    """生成事件指纹，用于识别同一活动的不同推广"""
    import hashlib
    
    # 提取核心关键词
    found_terms = [kw for kw in core_keywords if kw in text]
    
    # 提取并归一化日期
    dates = extract_normalized_dates(text)
    
    # 提取地点
    locations = extract_locations(text)
    
    # 生成MD5指纹
    fingerprint_str = "|".join(sorted(found_terms)) + "|" + "|".join(sorted(dates)) + "|" + "|".join(sorted(locations))
    return hashlib.md5(fingerprint_str.encode()).hexdigest()[:12]
```

**聚合策略：**
- 时间窗口：7天内相似事件归为同一活动
- 相似度判断：指纹匹配 + 关键词重叠率 > 60%
- 更新机制：同一事件的多条帖子聚合，保留最新信息

## 五、四大增值业务详解

> **排除方向**：团购、代切煤（涉及供应链、物流、关税、假货风险，不适合个人开发者）。

> **核心逻辑**：复用现有 AI 文案生成能力，封装成 SaaS 工具，面向谷圈/二游内容创作者收费。
> 目标用户（抖音/小红书创作者）与产品现有用户高度重合，竞品少，垂直赛道几乎没有专业工具。

> **合规边界**：不做票务交易，只做情报提醒。提醒工具不涉及支付，合规风险低。

#### 技术实现路径

1. 复用现有微博爬虫，新增展会/限购关键词过滤
2. 接入微信模板消息推送（需公众号）
3. 新增展会数据源：CP、BW、ChinaJoy 官网及官方微博

### 5.3 专业数据服务 ⭐⭐

> **核心逻辑**：面向 B 端（IP 方/谷子品牌方/同人社团/创作者工作室），提供谷圈舆情和热度数据。
> B 端付费意愿强，但需要先有足够大的用户基数再推。

#### 功能规划

| 功能 | 说明 | 定价参考 |
|------|------|---------|
| IP 热度榜单 | 每周/每月各 IP 热度排行榜（微博互动/搜索量） | 年度授权 999-2999 元 |
| 品牌舆情监控 | 特定品牌/IP 的全网口碑监控报告 | 年度授权 1999-4999 元 |
| 竞品对标分析 | 多个同类账号/IP 的数据对比 | 年度授权 1499-3999 元 |
| 白标工具 | 给工作室/同人社团提供专属管理后台 | 定制项目制 |
| 实时热词监控 | 谷圈热点词汇实时追踪 + 趋势图 | 月费 99-299 元 |

#### 技术实现路径

1. 利用现有爬虫数据积累行业数据集
2. 新增数据分析服务（热度计算、趋势分析）
3. 新增 B 端管理后台（独立于现有 C 端后台）
4. 按账号/按年付费，接入对公转账（工商个体户后）

---

### 5.4 社群情报引流 ⭐⭐⭐

> **核心逻辑**：不直接变现，做引流工具。通过情报分发在外部平台（微信/微博/抖音）吸引目标用户，引导进入小程序完成变现闭环。

#### 情报分类体系

| 类别 | 关键词示例 | 推送渠道 |
|------|-----------|---------|
| 新品情报 | "新品预售"、"现货发售"、"再版通知" | 公众号 + 微信群 |
| 限购情报 | "限时"、"抽选"、"一人一单"、"sold out" | 公众号 + 微信群 |
| 展会情报 | "漫展"、"CP"、"BW"、"ChinaJoy"、"摊位" | 公众号 + 小程序 |
| 二手行情 | "好价"、"好价出"、"捆出"、"求好价" | 微信群 |
| 活动情报 | "repo"、"返图"、"官周活动"、"联名" | 抖音 + 小红书 |

#### 技术实现路径

| 功能 | 说明 |
|------|------|
| 微信公众号接入 | 菜单栏跳转小程序，关键词自动回复 |
| 微博监控增强 | 新增限购/新品/展会关键词过滤，触发推送 |
| 微信群推送机器人 | 接入微信群，自动推送情报（需防封策略） |
| 抖音/小红书内容号 | 定期发布术语科普/展会预告，引导搜索小程序 |
| 推送去重与聚合 | 避免同一信息重复推送给用户 |
| 用户推送偏好管理 | 用户自主选择接收哪些类型情报 |

> **防封策略**：社群推送不做主动群发，优先做"用户查询 → 机器人回复"模式，降低封号风险。

### 5.5 变现架构总览

```
                    ┌─ 创作者工具包（直接变现）
                    │
引流层 ─────────────┼─ 展会资讯订阅（直接变现）
（社群情报）         │
                    ├─ 专业数据服务（B端变现）
                    │
                    └─ 小程序商品返佣（持续变现）
                           （阿里妈妈 CPS）
```

#### 各业务启动顺序

| 阶段 | 时间 | 业务 | 前提条件 |
|------|------|------|---------|
| 第一批 | 近期 | H5 落地 + 热点追踪系统 | 管理后台完成 |
| 第二批 | H5 上线后 | 社群情报引流体系 | H5 MVP 完成 |
| 第三批 | H5 + 小程序后 | 小程序商品返佣（CPS） | 小程序上线 + 用户积累 |
| 第四批 | 用户过万后 | 创作者工具包 | 小程序稳定运营 |
| 第五批 | 用户过5万后 | 展会资讯订阅 | 数据积累充分 |
| 第六批 | B端需求出现后 | 专业数据服务（B端） | 品牌方主动联系 |

> **注意**：H5 无需审核、即时上线、可多渠道传播，快速验证产品价值后再启动小程序开发。
